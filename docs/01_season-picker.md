# 01 — 계절 테마 선택 (Season Picker)

> 상태: **승인 · 구현 완료 (M1+M2)** · 2026-07-14

## 배경

정원의 계절은 접속 날짜로 자동 결정된다(`lib/season.ts`). 이것은 "살아있는 정원"의
핵심이지만, 특정 계절의 정원에 애착이 생긴 사용자가 그 계절에 머무를 방법이 없다.
계절별 표현(팔레트 8종·입자 4종)이 이미 전부 `html[data-season]` CSS로 구현되어
있어, 선택 기능은 "어느 계절 값을 찍을지"의 문제로 축소된다 — 지금이 가장 싸게
만들 수 있는 시점이다.

## 목표 / 비목표

**목표**

- 기본값은 지금과 동일한 자동(날짜 기반). 사용자가 원하면 계절 하나를 고정할 수 있다.
- 선택 즉시 정원 전체(팔레트·입자·다크 톤)가 그 계절로 바뀐다 — 시트 뒤로 보이는
  정원이 곧 미리보기다.
- 선택은 localStorage에 영속되어 다음 방문에도 유지된다.

**비목표**

- 자동 모드의 실시간 계절 갱신(자정·월 경계를 열어둔 채 넘는 경우)은 지금처럼
  다음 방문에 반영되는 것으로 충분하다.
- 과거 달 정원(기록 페이지)을 "그 달의 계절"로 그리는 것 — 별개 기능이며 이번
  범위가 아니다.
- 새 팔레트·새 입자·계절별 신규 연출 없음. 기존 8조합 값을 그대로 쓴다.
- 다크모드 수동 선택 없음 (시스템 설정 따름 유지).

## 계약

### localStorage (lib/storage)

| 키 | 값 | 의미 |
|---|---|---|
| `study-garden:season` | `'spring' \| 'summer' \| 'autumn' \| 'winter'` | 고정된 계절 |
| (키 없음) | — | 자동 (날짜 기반, 기존 동작) |

- "자동"은 `'auto'` 값을 저장하지 않고 **키 삭제**로 표현한다 — 기존 사용자
  (키 없음)와 자동 선택 사용자가 같은 상태가 되어 마이그레이션이 없다.
- 읽기/쓰기는 기존 repository 패턴(`lib/storage/`)에 함수로 추가한다.

### 상태 (stores)

- `season-store` 신설: `{ preference: Season | null, hydrated, hydrate(), setPreference() }`.
  기존 store 관례(hydrate에서만 localStorage 읽기)를 따른다.
- `SeasonProvider`는 `preference ?? seasonOf(new Date())`를 `data-season`에 찍는다.
  preference 변경 시 즉시 재적용 (useEffect 의존성).

### UI

- 헤더 설정 아이콘 → 기존 `DurationSheet`를 **설정 시트로 확장**한다:
  제목 "집중 시간" → "설정", 섹션 2개 — "집중 시간"(기존 목록 그대로),
  "정원의 계절"(자동 · 봄 · 여름 · 가을 · 겨울, 5행 목록 + 체크 표시).
- 파일은 `components/timer/DurationSheet.tsx` → `components/SettingsSheet.tsx`로
  이동·개명 (타이머 전용이 아니게 되므로).
- 카피(해요체 정원 톤, 초안 — 구현 시 확정):
  - 섹션 라벨: "정원의 계절"
  - 자동 행: "계절 따라 · 지금은 여름" (현재 계절을 함께 표기)
  - 보조 문구(계절 고정 시): "정원이 이 계절에 머물러요."

### 충돌 여부

기존 계약 문서 없음(이 문서가 01). `study-garden:sessions`, `study-garden:duration`
키와 독립적이며 스키마 충돌 없음.

## 마일스톤

- **M1 — 저장·상태·적용**: storage 함수 + season-store + SeasonProvider 연동.
  완료 기준: 콘솔에서 `localStorage` 값을 바꿔 새로고침하면 4계절이 각각 적용되고,
  키 삭제 시 자동(현재 여름)으로 돌아온다 — 스크린샷으로 확인.
- **M2 — 설정 시트 UI**: 시트 확장 + 카피 + 즉시 전환.
  완료 기준: 시트에서 계절 선택 → 새로고침 없이 정원 전환, 재방문 시 유지.
  시트 라이트/다크 + 고정된 계절 표현 스크린샷, `npx next build` 통과,
  DoD 체크리스트(하드코딩 grep·해요체·reduced-motion) 전부 통과.

## 측정 계획

임계값·방식 선택이 필요한 수치 결정은 없다. 유일한 감각 판정은 **계절 전환 순간의
팔레트 점프**다 — M2에서 실제 브라우저로 전환을 보고, 거슬리면 `html`의 색 전환
transition(기존 토큰만 사용) 추가 여부를 실측 화면과 함께 옵션으로 보고하고 사용자가
결정한다 (추측으로 미리 넣지 않는다).

## 알려진 리스크 / 한계

- **첫 페인트 계절 플래시**: SSG 기본은 봄 팔레트, 클라이언트에서 저장된 계절로
  전환된다. 기존 자동 모드도 동일하게 동작해 온 구조라 신규 리스크는 아니다.
  → 다음 단계 영향: 없음 (기존 동작 유지).
- **"지금은 여름" 표기**: 시트는 클라이언트 렌더라 하이드레이션 안전하지만,
  날짜 기반 문구가 컴포넌트에 처음 들어온다. `seasonOf(new Date())`를 렌더 중
  직접 호출하지 않고 store 경유로 통일한다.
  → 다음 단계 영향: M2 구현 시 하이드레이션 규칙 준수 확인 필요.
- **(구현 후 관찰) 시트 뒤 미리보기는 모바일에서 제한적**: 430px 뷰포트에서는 시트가
  정원 대부분을 가리고 스크림이 남은 부분을 어둡게 해, "선택 즉시 정원이 미리보기로
  보인다"는 효과가 약하다. 시트를 닫는 순간 결과가 보이므로 동작 자체는 문제없다.
  → 다음 단계 영향: 불만 피드백이 있으면 V2에서 계절 행에 미니 팔레트 스와치 추가 검토.
- **계절 고정을 잊는 문제**: 겨울에 정원을 여름으로 고정해 두면 "살아있는 정원"
  감각이 약해질 수 있다. V1 범위에서는 자동 행에 현재 계절을 상시 표기하는 것으로
  대응하고, 리마인드성 연출은 만들지 않는다.
  → 다음 단계 영향: 사용 피드백에 따라 V2에서 재검토.
